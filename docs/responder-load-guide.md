# Responder Load Guide

The responder-load report shows which owner is absorbing the most forecasted support pressure.

Use it to answer:

- who currently owns the most support effort
- whether breaches are concentrated on one owner
- which owners are mostly handling on-track work versus risky work

The pressure index is intentionally simple:

- `effort_points`
- plus `3x` each breach
- plus `1x` each at-risk item

That makes breach-heavy ownership float to the top even when total effort is similar.
